INFO PROJECT WEB

Berikut ini adalah ketentuan untuk Project Web :

1. Project web tidak ada hubungannya dengan UI yang di sempurnakan hasil dr UX
2. Project web melanjutkan UI yang sudah di rancang di semester 1 (silakan klo ada upgrade)
3. Masing-masing anggota minimal membuat :
> setiap anggota membuat 2 crud (input, edit, delete, select) full PHP
> setiap anggota membuat 2 endpoint dengan minimal (2 method / endpoint -> POST, GET, PUT, DELETE)
> role admin / pengelola dan sejenisnya dibuat full PHP
> role lainnya seperti konsumen / penjual / dan sejenisnya bisa dibuat frontend - backend
> frontend menggunakan HTML, CSS, JS atau React (include HTML CSS) - Pilih salah satu
> backend menggunakan full PHP Native
> aplikasi mengimplementasikan otorisasi dan authentikasi baik di PHP full (role admin dll) ataupun akses API PHP dng menggunakan API Key
> Fungsionalitas yang dibuat cukup fungsionalitas utama saja yang didahulukan yang penting memenuhi spek yang ditentukan (banyaknya jumlah CRUD dan Endpoint)
4. Saat assessment 3 semua harus di gabungkan, silakan bisa menggunakan Github dalam pengerjaan project
5. GIthub harusnya sudah diajarkan di mata kuliah SIM
6. Assessment 3 akan menguji codingan yang di buat
